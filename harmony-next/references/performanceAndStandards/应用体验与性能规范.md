# 应用体验与性能规范 (UX/Performance)

本文为基于 HarmonyOS 官方文档整理的主题导读，聚合应用体验与性能规范相关要点。

华为提出了“元服务”与“应用”的高性能开发规范，重点在于提升响应速度和流畅度，减少感知延迟。

## 1. 延迟优化 (Latency Optimization)
*   **启动优化**：
    *   减少 `onCreate` 和 `onWindowStageCreate` 中的同步耗时操作。
    *   推荐使用 **AppStartup** 框架异步初始化 SDK 及相关组件。
*   **响应优化**：
    *   **主线程解放**：绝对避免在 UI 线程（主线程）执行 I/O 操作或复杂的数据计算。
    *   **异步任务**：使用 `TaskPool` 或 `Worker` 将耗时逻辑转移到后台任务执行。

## 2. 帧率优化 (Frame Rate Optimization)
*   **布局扁平化**：
    *   减少组件嵌套层级，优先使用 `RelativeContainer` 代替深层的 `Row`/`Column` 嵌套。
*   **按需加载与复用**：
    *   对于长列表，务必使用 `LazyForEach`，避免一次性渲染大量节点。
    *   结合 `@Reusable` 装饰器实现组件级复用，降低创建和销毁的开销。
*   **动效优化**：
    *   优先使用系统提供的**属性动画 (Property Animation)**。
    *   避免在每一帧通过 `setInterval` 或 `requestAnimationFrame` 手动修改状态变量。

## 3. 资源使用优化 (Resource Management)
*   **内存优化**：
    *   及时释放不再使用的资源（如大图、不再活跃的事件监听器）。
    *   对于图片密集型应用，建议使用合适的占位图并配置缓存（如使用图片库能力）。
*   **CPU/功耗优化**：
    *   避免无效的定时器和高频轮询。
    *   **后台规范**：在应用进入后台（`onBackground`）时，主动挂起或销毁非必要的后台任务和动画，以降低功耗。

## 4. UX 体验建议
*   **骨架屏与过渡**：在数据未加载完成前，提供合适的骨架屏 (Skeleton) 或 Loading 动画，缓解用户等待焦虑。
*   **一致性**：遵循 HarmonyOS 的设计规范，确保返回手势、交互动效的系统一致性。
