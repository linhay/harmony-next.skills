# 性能调优与 Profiler 指南

本文为基于 HarmonyOS 官方文档整理的主题导读，聚合性能分析与调优工具相关要点。

HarmonyOS 提供了 **DevEco Profiler** 作为核心性能调优工具，通过 Insight Session 进行深度分析。

## 1. CPU 活动分析 (CPU Profiler)
用于识别代码中的性能瓶颈（如耗时函数、高负载线程等）。
*   **核心指标**：CPU 使用率、线程状态（Running, Sleeping, Blocked）、调用栈（Call Tree）。
*   **关键功能**：
    *   **Time Line**：展示 CPU 利用率、线程调度情况及各线程的执行序列。
    *   **Call Tree（调用树）**：按层级展示函数调用关系，帮助快速定位“热点函数”。
    *   **Flame Chart（火焰图）**：直观展示函数调用的深度和耗时占比。
*   **分析场景**：主线程卡顿定位、后台线程高负载分析、低效算法排查。

## 2. 内存分析 (Memory Profiler)
用于检测内存泄漏、内存抖动及查看对象分配详情。
*   **核心指标**：Heap Size（堆内存）、Native Memory、Graphics Memory。
*   **关键功能**：
    *   **Allocation Tracking**：实时追踪对象的分配和销毁。
    *   **Snapshot (内存快照)**：对比不同时间点的内存快照，精准定位内存增长点和内存泄漏（Memory Leak）。
    *   **GC 监控**：观察垃圾回收频率及耗时，避免因频繁 GC 导致的卡顿。

## 3. 帧率分析 (Frame Analysis)
专门用于分析 UI 丢帧、卡顿，查看每帧的渲染耗时。
*   **核心指标**：实时帧率（FPS）、丢帧数（Jank）、每帧渲染耗时。
*   **关键功能**：
    *   拆解每一帧的渲染流水线（UI 线程 -> 渲染线程 -> GPU）。
    *   **卡顿定位**：自动识别超过 16.6ms（60Hz）或 8.3ms（120Hz）的帧，并关联到具体的组件更新或逻辑执行。

## 4. 启动分析 (Launch Analysis)
针对应用启动过程进行专项录制，优化冷启动和热启动速度。

## 5. 网络与耗电量分析
*   **网络分析 (Network)**：监控 HTTP/HTTPS 请求，查看请求头、响应体及传输耗时。
*   **耗电量分析 (Energy)**：分析应用在运行过程中的各项资源（如网络、GPS、传感器）对电量的消耗。

## 6. 其他调优工具
*   **HiChecker**：在代码运行时自动检测潜在的性能问题（如主线程执行耗时操作）。
*   **HiTrace**：用于跨进程、跨设备的调用链追踪和性能分析。
