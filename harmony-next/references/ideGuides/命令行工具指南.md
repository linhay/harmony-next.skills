# 命令行工具指南

本文为基于 HarmonyOS 官方文档整理的主题导读，聚合 `hdc`、`ohpm`、`hvigor` 等命令行工具相关要点。

HarmonyOS 命令行工具（Command Line Tools）支持脱离 IDE 进行自动化构建、安装和调试。

## 1. 工具获取与安装
### 获取方式
*   **DevEco Studio 内部获取**：`Settings > SDK > Tools` 勾选 `Command Line Tools` 下载。
*   **独立获取**：从华为开发者联盟官网下载独立软件包。

### 环境变量配置
为了在全局使用命令行，需将 `bin` 目录添加到系统 PATH：

*   **hdc**: SDK 路径下的 `toolchains` 目录。
*   **ohpm**: 独立安装包下的 `bin` 目录。

```bash
# 以 macOS 为例
export PATH=$PATH:/path/to/sdk/toolchains
export PATH=$PATH:/path/to/ohpm/bin
```

## 2. 核心命令行工具
### hdc (HarmonyOS Device Connector)
连接真机或模拟器的调试桥，类似 adb。
*   `hdc list targets`: 列出在线设备。
*   `hdc shell`: 进入设备 shell。
*   `hdc install [file.hap]`: 安装应用。
*   `hdc file send [local] [remote]`: 推送文件。
*   `hdc hilog`: 查看设备实时日志。

### ohpm (OpenHarmony Package Manager)
三方库管理工具。
*   `ohpm install`: 安装 `oh-package.json5` 中的依赖。常用 `--all` 在多模块项目中安装所有依赖。
*   `ohpm publish [package_path]`: 发布三方库。
*   `ohpm update`: 更新依赖版本。
*   `ohpm config set registry [url]`: 设置私有仓地址。

### hvigor (Build Tool)
基于 JS 的构建引擎，负责编译、打包、签名。
*   `./hvigorw assembleHap`: 编译生成 HAP 包。
*   `./hvigorw assembleApp --mode release`: 编译生成上架用的 APP 包（Release 模式）。
*   `./hvigorw clean`: 清理构建缓存。
*   参数 `-p product`: 指定构建产物（Product）。
*   参数 `--no-daemon`: CI 环境建议开启，避免常驻进程。

### CodeLinter (代码检查工具)
用于 ArkTS/TS 代码静态扫描，检查规范与缺陷。
*   路径：`SDK/toolchains/codelinter`。
*   `codelinter -p [project_path]`: 执行扫描。
*   参数 `-o [path]`: 指定报告输出路径（支持 JSON, HTML）。
*   参数 `-c [config]`: 指定自定义规则配置文件。

### hstack (堆栈反解析工具)
将混淆后的崩溃堆栈还原为源码行号。
*   路径：`SDK/toolchains/hstack`。
*   `hstack --source-map [map_file] --stack [stack_file]`: 执行反解析。

### Emulator (模拟器管理工具)
位于 DevEco Studio 安装目录下的 `tools/emulator` 目录，用于通过命令行管理本地模拟器。
*   `Emulator -list -details`: **（核心）** 列出本地所有模拟器实例的详细信息，获取启动所需的 `name`、`path` 和 `imageRoot`。
*   `Emulator -hvd [Name] -path [Path] -imageRoot [ImageRoot]`: **（启动）** 启动指定模拟器。
    *   可选参数 `-hdcport [Port]`: 指定 hdc 连接端口（10000-16555）。
*   `Emulator -stop [Name]`: **（停止）** 停止并关闭模拟器。
*   `Emulator -version` / `-help`: 查看版本或完整帮助文档。
*   `Emulator -logZip [Name] -logPath [ZipPath]`: 收集模拟器日志并打包。
*   **注意**：命令行不支持创建模拟器实例，必须先在 IDE 的 Device Manager 中手动创建。

#### 实操案例：macOS 环境下查找与启动模拟器
如果你在终端直接输入 `Emulator` 提示找不到命令，可以按照以下实操流程进行：

1.  **定位工具路径**：
    默认路径通常为：`/Applications/DevEco-Studio.app/Contents/tools/emulator/emulator`。
2.  **查看本地已创建的模拟器**：
    执行以下命令获取实例的详细 JSON 信息：
    ```bash
    /Applications/DevEco-Studio.app/Contents/tools/emulator/emulator -list -details
    ```
3.  **提取启动参数**：
    从返回的 JSON 中找到以下三个核心字段：
    - `"name"`: 模拟器名称（如 `"nova 15 Pro"`）。
    - `"instancePath"`: 实例存储的具体路径。
    - `"imageRoot"`: SDK 镜像根目录。
4.  **执行启动命令**：
    ```bash
    /Applications/DevEco-Studio.app/Contents/tools/emulator/emulator \
      -hvd "你的模拟器名称" \
      -path "你的实例具体路径" \
      -imageRoot "你的镜像根目录"
    ```
    *   **注意**：`-path` 参数必须指定为 `Emulator -list -details` 命令返回的 `instancePath` 完整路径。如果路径包含空格，务必使用双引号。
    *   **限制**：命令行工具仅支持管理本地已创建的模拟器生命周期（启动/停止），不支持通过命令行创建模拟器或进行数据清除（Wipe Data）。

#### 模拟器应用管理 (使用 hdc 工具)
模拟器启动成功后，管理其内部应用需使用 `hdc` 工具：
*   **列出已安装应用**：`hdc shell bm dump -a`
*   **安装新应用**：`hdc install [path_to_hap]`
*   **卸载应用**：`hdc uninstall [bundle_name]`
*   **运行/拉起应用**：`hdc shell aa start -b [bundle_name] -a [ability_name]`

### ArkGuard (混淆工具)
用于源码混淆，提升代码安全性。
*   通过 `obfuscation-rules.txt` 配置。
*   支持名称混淆、属性混淆。

## 3. 命令行构建 APP 流程 (CI/CD)
在 CI/CD 环境下脱离 IDE 完成构建的全流程：

1.  **环境准备**：配置 Node.js、JDK 及 `HOS_SDK_HOME` 环境变量。
2.  **安装依赖**：执行 `ohpm install`。建议配置缓存目录加速。
3.  **签名配置**：在 `build-profile.json5` 中配置签名，或通过 Secret 变量注入证书（.p12）及密码。
4.  **执行构建**：使用 `./hvigorw clean assembleApp --mode release`。
5.  **质量检查**：执行 `codelinter` 扫描代码，确保 `errorCount` 为 0。
6.  **结果处理**：获取 `build/outputs/app/release/` 下的 APP 包，并结合崩溃日志使用 `hstack` 进行监控。
