# 调试与日志分析指南

本文为基于 HarmonyOS 官方文档整理的主题导读，聚合调试、日志与故障分析相关要点。

HarmonyOS 提供了强大的调试与日志分析工具，帮助开发者定位代码逻辑问题。

## 1. 调试器 (Debugger)
DevEco Studio 内置基于 LLDB 的调试器，支持单步执行、变量观察等功能。

### 核心操作
*   **断点 (Breakpoint)**：在行号旁点击即可添加。
*   **条件断点 (Conditional Breakpoint)**：右击断点并设置条件表达式。
*   **日志断点 (Logpoint)**：触发时不停止，仅在控制台打印日志。
*   **执行控制**：
    *   **F8 (Step Over)**：单步跳过。
    *   **F7 (Step Into)**：单步跳入。
    *   **Shift+F8 (Step Out)**：单步跳出。
    *   **F9 (Resume Program)**：恢复运行。

### 变量观察
*   **Variables 窗口**：实时查看作用域内变量的值。
*   **Evaluate Expression**：在调试期间动态执行代码片段。

## 2. 日志分析 (Log & Fault Analysis)
日志系统是定位问题的主要手段，分为系统日志和应用日志。

### Log 窗口
*   **过滤级别**：DEBUG, INFO, WARN, ERROR, FATAL。
*   **过滤条件**：按 Tag、进程、关键词过滤。
*   **实时查看**：支持连接设备后实时滚动查看。

### 应用日志打印 (ArkTS)
```typescript
import hilog from '@ohos.hilog';

const DOMAIN = 0x0001;
const TAG = 'MyApplication';

hilog.info(DOMAIN, TAG, 'User logged in: %{public}s', 'username');
```
*   `%{public}s`：明文显示，适合调试。
*   `%{private}s`：发布版本自动脱敏。

### 故障分析 (FaultLog)
*   **Crash 分析**：应用崩溃时，系统会自动生成 FaultLog。
*   **堆栈反解析**：DevEco Studio 支持导入崩溃堆栈文件并自动关联源码。
*   **hstack 工具**：命令行工具，用于将混淆或地址化的堆栈还原为源码行号。
    *   命令：`hstack --source-map [map_file] --stack [stack_file]`。
