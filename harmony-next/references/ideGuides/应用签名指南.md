# 应用/元服务签名指南

本文为基于 HarmonyOS 官方文档整理的主题导读，聚合签名、证书与发布配置相关要点。

应用签名是确保应用完整性和来源可信的核心机制。DevEco Studio 提供了**自动签名**和**手动签名**两种方案。

## 1. 核心概念
*   **密钥 (Key, .p12)**：包含非对称加密的公钥和私钥，存储在密钥库文件中，用于数字签名和验证。
*   **证书请求文件 (CSR, .csr)**：包含公钥及开发者信息，用于向 AppGallery Connect (AGC) 申请数字证书。
*   **数字证书 (Digital Certificate, .cer)**：由华为 AGC 颁发，证明开发者的身份。
*   **Profile 文件 (.p7b)**：包含应用包名、数字证书信息、允许申请的权限列表（ACL）以及允许调试的设备列表。
*   **App ID**：应用在华为生态中的唯一标识，由包名（Bundle Name）和 AGC 分配的 ID 组成。

## 2. 操作步骤
### 自动签名（推荐调试使用）
1.  **连接设备**：连接真机或启动模拟器。
2.  **进入配置**：`File > Project Structure > Project > Signing Configs`。
3.  **登录账号**：点击 "Sign In" 登录华为账号。
4.  **选择模式**：
    *   **Associate with registered application**：关联 AGC 已注册应用，支持开通推送、地图等开放能力。
    *   **Automatically generate signature**：不关联 AGC 应用，仅用于快速调试。
5.  **完成**：点击 "OK"，系统自动生成并配置签名文件。

### 手动签名（发布或特殊调试场景）
1.  **生成密钥和 CSR**：通过 `Build > Generate Key and CSR` 生成 `.p12` 和 `.csr` 文件。
2.  **AGC 申请**：在 AGC 后台上传 CSR，申请调试证书（.cer）和 Profile 文件（.p7b）。
3.  **本地配置**：在 `Signing Configs` 中取消自动签名，手动指定 `.p12`、`.cer` 和 `.p7b` 文件路径及密码。

## 3. 常见问题
*   **ACL 权限报错**：如果应用使用了受限权限（如 `ohos.permission.READ_CONTACTS`），自动签名可能失败，需在 AGC 手动申请 ACL 权限。
*   **时间不一致**：本地系统时间必须与北京时间同步，否则签名会校验失败。
*   **包名冲突**：自动签名时若 AGC 已存在同包名但不同团队的应用，会提示冲突。
