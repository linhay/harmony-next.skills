# 独立命令行工具配置手册 (Standalone CLI)

本手册针对 CI/CD 环境及纯命令行开发场景，提供 **HarmonyOS NEXT 独立命令行工具包** 的下载、安装与深度配置指南，解决 IDE 内嵌工具常见的环境依赖报错问题。

## 1. 工具获取 (Download)

在自动化构建服务器（如 Jenkins/GitLab Runner）上，推荐使用独立工具包而非从 DevEco Studio 拷贝。

*   **官方下载地址**：[华为开发者联盟下载中心 - Command Line Tools](https://developer.huawei.com/consumer/cn/download/command-line-tools-for-hmos)
*   **版本选择**：请确保下载的工具包版本与项目使用的 SDK API 版本一致。

## 2. 安装与环境配置 (Setup & Env)

解压工具包到指定目录（例如 `/opt/huawei/command-line-tools`），并配置以下环境变量：

### 核心环境变量
在 `~/.bash_profile` 或 `~/.zshrc` 中添加：

```bash
# 1. 独立工具包根目录
export HOS_SDK_HOME=/path/to/command-line-tools/sdk

# 2. OpenHarmony SDK 路径 (关键：解决 SDK component missing)
export OHOS_BASE_SDK_HOME=$HOS_SDK_HOME/openharmony

# 3. HarmonyOS (HMS) SDK 路径
export DEVECO_SDK_HOME=$HOS_SDK_HOME/harmonyos

# 4. 将各工具 bin 目录加入 PATH
export PATH=$PATH:$HOS_SDK_HOME/openharmony/toolchains  # hdc
export PATH=$PATH:/path/to/command-line-tools/ohpm/bin # ohpm
export PATH=$PATH:/path/to/command-line-tools/hvigor/bin # hvigorw
```

执行 `source` 使其生效。

## 3. 登录与授权 (Login & Auth)

### ohpm 登录
发布三方库到华为公有仓或私有仓前，需进行登录：
```bash
ohpm login
```
该命令会引导至浏览器完成华为账号授权，并生成 `~/.ohpm/.ohpmrc` 配置文件。

### hdc 授权
首次连接真机或模拟器时，需在设备端点击“允许 USB 调试授权”。命令行验证方式：
```bash
hdc list targets
```
若状态显示为 `unauthorized`，请重新插拔设备并同意授权弹窗。

## 4. 项目配置规范 (Project Config)

独立命令行工具对项目配置文件的严谨性要求极高，请务必核对以下三点：

### A. 版本的字符串化 (SDK Version)
在 `build-profile.json5` 中，SDK 版本号**必须使用字符串**，严禁使用纯数字。
*   ✅ **正确**：`"compileSdkVersion": "5.0.0(12)"`
*   ❌ **错误**：`"compileSdkVersion": 12`

### B. modelVersion 一致性
确保项目根目录的 `oh-package.json5` 与 `hvigor-config.json5` 中的 `modelVersion` 字段保持一致（例如均设置为 `"5.0.0"`）。

### C. 缺失配置文件补齐
如果项目是从其他平台迁移或手动创建，必须确保以下文件存在，否则 `hvigor` 无法初始化：
*   项目根目录：`hvigorfile.ts`, `build-profile.json5`, `hvigor-config.json5`。
*   模块目录：`hvigorfile.ts`, `build-profile.json5`。

## 5. 常见问题排查 (FAQ)

*   **报错：`SDK component missing`**
    *   检查 `OHOS_BASE_SDK_HOME` 是否正确指向了工具包内的 `openharmony` 目录。
*   **报错：`Command not found`**
    *   检查工具包下的 `ohpm/bin` 和 `hvigor/bin` 路径是否正确合并到了 `PATH`。
*   **构建产物路径**
    *   命令行构建的产物默认位于 `[module_root]/build/outputs/`。
